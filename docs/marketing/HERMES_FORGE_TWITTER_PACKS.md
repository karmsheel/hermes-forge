# Hermes Forge — Twitter / X Tweet Packs

Ready-to-post copy for the **HermesForge** account.  
Product spine: agent-native business process mapping studio — brief → live diagram + process model → critique → forge → automate — powered by **Hermes Agent**, **local-first / BYOK**, desktop-first. Rooms: **Foundation → Map → Monitor → Automate**. Workshop = forge bench inside Map.  
Site: [hermes-forge.com](https://www.hermes-forge.com/)

## How these packs were shaped

High-engagement patterns from recent software/dev product posts on X:

| Pattern | Use |
|---------|-----|
| One-job product line | Clear positioning |
| Problem → reframe | Pain without feature dump |
| Scannable bullets / lists | Saves and bookmarks |
| Local-first / BYOK trust | Differentiation |
| Ship notes with specifics | Credibility |
| Short metaphors | Memorability |
| Demo prompts / how-to | Replies and trials |

**Media:** Prefer screenshot, GIF, or 15–40s clip on almost every post.  
**Cadence:** ~1/day (or every other day). Minimal hashtags. Reply to your own posts with a short clip when a post takes off.

```
Hermes Forge is for operators who are tired of being the process.

Get a living process map.
```

### C2 — The real SOP problem

```
SOPs fail in three ways:

1. Never written
2. Written once, wrong forever
3. Written correctly, never opened again

Hermes Forge targets a fourth path:
a map you critique in conversation and keep current as the business changes.
```

### C3 — Handoffs

```
Most “ops problems” are handoff problems.

Sales → success
Success → product
Finance → whoever chases the invoice
Support → engineering with a screenshot and a prayer

Hermes Forge maps handoffs as first-class flow — not a bullet list buried in a wiki.
```

### C4 — Hero dependency

```
If the business only works when one person is in Slack…

you don’t have a process.

Operators use Hermes Forge to pull that knowledge out of one head and into a shared map the team can see, argue with, and improve.
```

### C5 — Board / consultant graveyard

```
The last process project produced:
• a deck
• a RACI nobody opens

Hermes Forge is built so the output is a living map inside the studio —
not a PDF that ages out the week after the workshop.
```

### C6 — Clarity before headcount

```
Hiring before you map the work multiplies chaos.

Operators know this.
```

### C7 — “How we do things here”

```

Hermes Forge aims the map at what you actually do — then you decide what to document, measure, and automate.
```

### C17 — Weekly ops ritual

```
A simple Forge ritual for operators:

Every Friday:
1. Open the process that broke this week
2. Update the map in chat (“we actually do X, not Y”)

Thirty minutes. Map stays honest. Less tribal knowledge rot.
```
